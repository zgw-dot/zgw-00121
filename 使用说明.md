# 门店交接班差异登记工具 - 使用说明

## 概述

本工具用于门店收银员与值班长之间的交接班现金差异登记、复核与关闭流程，支持完整的操作留痕、撤销追踪、CSV 导入导出，所有数据本地持久化，重启不丢失。

## 快速启动

### 方式一：双击启动（推荐）
1. Windows 双击 `启动.bat`
2. Linux / macOS 执行 `bash 启动.sh`

### 方式二：命令行启动
```bash
pip install -r requirements.txt
python app.py
```

启动成功后访问：**http://127.0.0.1:5000/**

## 业务流程

### 标准流程
```
收银员新建 → 提交复核(待复核) → 值班长复核通过 → 值班长关闭(已关闭)
                              ↘ 退回(已退回) → 收银员补充备注 → 重新提交
```

### 状态说明
| 状态     | 英文标识   | 说明                                           |
|----------|------------|------------------------------------------------|
| 草稿     | draft      | 新建未提交，仅创建人可见可改                   |
| 待复核   | pending    | 已提交，等待值班长处理                         |
| 复核通过 | reviewed   | 值班长已通过，可关闭                           |
| 已退回   | returned   | 被退回，收银员需补充备注后重提                 |
| 已关闭   | closed     | 流程完成，不可修改                             |
| 已撤销   | revoked    | 被撤销，自动生成新草稿用于更正，历史保留       |

### 撤销机制（核心）
> **任何更正都必须通过"撤销"来完成，禁止直接修改已关闭/已复核的单据。**

1. 值班长/管理员点击「撤销」→ 填写撤销原因
2. 原单据状态变为「已撤销」，操作时间线记录撤销行为
3. 系统自动生成一张新的草稿（编号带 `-R` 后缀，保留原单关联）
4. 收银员对新草稿更正后重新走正常提交流程
5. 两张单据的时间线互相引用，审计链完整

- **状态筛选**：6 种状态单选

- **预警处置筛选**：按预警处置状态筛选（未处理 / 已确认 / 需跟进 / 已忽略）
- **预警记录筛选**：预警记录页面支持按处置状态独立筛选

## 差异汇总（首页顶部卡片）
- 待复核：数量 + 金额合计
- **未结数量**：草稿 + 待复核 + 已退回 + 复核通过 的总单数（核心指标）

## CSV 导入导出

### 导出字段
单据编号、状态、班次、班次日期、收银员、差异金额、原因、备注、创建人、创建时间、复核人、关闭人、退回说明、关闭说明、撤销人、撤销时间、关联原单、导入来源、预警规则、预警原因、处置状态、处置备注、处理人、处理时间

> 注：若一张单据有多条预警，会按预警拆分为多行展示，单据基础信息仅在第一行显示。

### 导入要求
- 编码：UTF-8（带 BOM 也支持）或 GBK
- 必填字段：单据编号、班次、班次日期、收银员、差异金额
- 负金额必须有原因
- 不允许导入已关闭或已撤销状态的单
- 不允许重复导入相同单号
- 支持中文表头（即导出格式可以直接再次导入）

## 异常预警机制

### 三种预警规则
| 规则类型              | 标识                  | 说明                                                   |
|-----------------------|-----------------------|--------------------------------------------------------|
| 单笔差异金额阈值      | `single_amount`       | 单笔单据差异金额绝对值 ≥ 阈值时告警                    |
| 收银员当日累计差异    | `cumulative_amount`   | 同一收银员当日所有单据的差异金额绝对值之和 ≥ 阈值时告警 |
| 连续退回次数          | `consecutive_return`  | 同一收银员状态为"已退回"的单据数 ≥ 阈值时告警           |

### 权限说明
- **值班长 / 管理员**：可创建、编辑、删除、启用/禁用预警规则，可导入/导出规则 CSV，可查看预警记录和操作日志
- **收银员**：**只能看到预警结果**（单据列表的预警标签、单据详情的预警原因、预警记录页面），**不能**看到预警规则列表、阈值、启用状态、创建人等配置信息，也不能对规则做任何改动

### 预警触发时机
预警在以下动作时自动检查：
- 收银员新建单据（保存草稿）
- 收银员提交复核
- CSV 导入单据

> **注意**：预警只做标记，**不阻断原有复核/关闭/撤销流程**。命中预警的单据仍可正常通过复核、关闭或撤销。

### 同单去重
同一单据在多次触发检查（如先存草稿再提交）时，同一条规则只会产生一条预警记录，不会重复写入。

### 预警处置闭环（核心功能）

预警产生后，值班长/管理员可对每条预警进行处置，形成完整的处理闭环。

#### 处置状态说明
| 状态     | 标识         | 说明                                               |
|----------|--------------|----------------------------------------------------|
| 未处理   | unprocessed  | 新产生的预警，尚未有人处理（默认状态）             |
| 已确认   | confirmed    | 已核实情况，按正常流程处理完毕                     |
| 需跟进   | follow_up    | 需要后续跟进处理，如转财务核查、联系当事人等       |
| 已忽略   | ignored      | 属于误报或无需处理的预警                           |

#### 值班长/管理员操作指南
1. **查看预警**：在「异常预警」→「预警记录」中查看所有预警，或在单据详情页查看该单的预警
2. **处置预警**：点击预警记录右侧的「处置」按钮（或单据详情中预警项旁的「处置」按钮）
3. **选择状态**：选择上述四种处置状态之一
4. **填写备注**：详细说明处理情况、跟进计划或忽略原因
5. **保存**：系统自动记录处理人、处理时间，并更新版本号

#### 处置冲突保护
如果同一条预警已被其他用户处理过，系统会提示冲突并显示最新状态，防止静默覆盖。此时请刷新页面查看最新情况后再决定是否需要重新处置。

#### 筛选功能
- 预警记录页面支持按「处置状态」筛选，快速定位未处理或需跟进的预警
- 单据列表支持按「预警处置」筛选，快速找到含有特定处置状态预警的单据

#### 收银员可见范围
- ✅ 可看到预警结果（预警标签、预警原因）
- ✅ 可看到处置状态、处置备注、处理人、处理时间
- ❌ 不能修改处置状态
- ❌ 不能看到预警规则配置（阈值、类型等）
- ❌ 不能看到操作日志

### 撤销后预警隔离
单据被撤销后生成的新单据会**重新触发预警检查并生成独立的预警记录**，原单的处置结果不会自动带到新单。新单的预警需要值班长重新处置，确保历史处理结果与新单完全隔离。

### 预警规则 CSV 导入/导出
- **导出**：UTF-8 编码，字段为「规则名称、规则类型、阈值、是否启用、描述」
- **导入**：
  - 同名规则**不会被覆盖**，而是直接跳过，并在导入结果中给出明细
  - 类型无效、阈值非正或缺失的行会判定为失败并给出原因
  - 每次导入（无论成功/跳过/失败）均写入操作日志

## 常见问题

**Q: 浏览器无法访问？**
A: 确认 `启动.bat` 窗口未关闭，且无端口冲突（默认 5000）。

**Q: 忘记密码？**
A: 使用 SQLite 工具（如 DB Browser for SQLite）打开 `data/shift_diff.db`，查看 `users` 表即可看到明文密码，或替换为新值。

**Q: 想恢复到上周五的数据？**
A: 停止服务 → 用上周五的备份 `shift_diff_YYYYMMDD.db` 覆盖当前文件 → 重启。

**Q: 如何新增用户？**
A: 管理员直接在 `users` 表中 INSERT：
```sql
INSERT INTO users (username, password, role, display_name)
VALUES ('cashier3', 'cashier123', 'cashier', '收银员小王');
```

**Q: 数据文件体积越来越大？**
A: 可定期导出 CSV 归档后，执行 `VACUUM` 命令压缩 SQLite 文件。
