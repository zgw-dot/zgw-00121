# 门店交接班差异登记工具 - 使用说明

## 概述

本工具用于门店收银员与值班长之间的交接班现金差异登记、复核与关闭流程，支持完整的操作留痕、撤销追踪、CSV 导入导出，所有数据本地持久化，重启不丢失。

## 快速启动

### 方式一：双击启动（推荐）
1. Windows 双击 `启动.bat`
2. Linux / macOS 执行 `bash 启动.sh`

### 方式二：命令行启动
```bash
pip install -r requirements.txt
python app.py
```

启动成功后访问：**http://127.0.0.1:5000/**

## 业务流程

### 标准流程
```
收银员新建 → 提交复核(待复核) → 值班长复核通过 → 值班长关闭(已关闭)
                              ↘ 退回(已退回) → 收银员补充备注 → 重新提交
```

### 状态说明
| 状态     | 英文标识   | 说明                                           |
|----------|------------|------------------------------------------------|
| 草稿     | draft      | 新建未提交，仅创建人可见可改                   |
| 待复核   | pending    | 已提交，等待值班长处理                         |
| 复核通过 | reviewed   | 值班长已通过，可关闭                           |
| 已退回   | returned   | 被退回，收银员需补充备注后重提                 |
| 已关闭   | closed     | 流程完成，不可修改                             |
| 已撤销   | revoked    | 被撤销，自动生成新草稿用于更正，历史保留       |

### 撤销机制（核心）
> **任何更正都必须通过"撤销"来完成，禁止直接修改已关闭/已复核的单据。**

1. 值班长/管理员点击「撤销」→ 填写撤销原因
2. 原单据状态变为「已撤销」，操作时间线记录撤销行为
3. 系统自动生成一张新的草稿（编号带 `-R` 后缀，保留原单关联）
4. 收银员对新草稿更正后重新走正常提交流程
5. 两张单据的时间线互相引用，审计链完整

### 2. 业务校验（非法路径拦截）
| 场景                                      | 结果       | 实现位置              |
|-------------------------------------------|------------|-----------------------|
| 负金额（短款）未填原因                    | 拒绝提交   | `validate_voucher_payload` / `api_submit_voucher` |
| 收银员关闭自己创建的单据                  | 拒绝关闭   | `api_close_voucher` L558 |

| CSV 重复导入已撤销的单号                  | 拒绝导入   | `api_import_csv` L740 |
| 被退回后未补充备注/原因即再次提交         | 拒绝提交   | `api_submit_voucher` L467-L474 |
| 退回未填退回说明                          | 拒绝退回   | `api_review_voucher` L530 |
| 撤销未填撤销原因                          | 拒绝撤销   | `api_revoke_voucher` L578 |
| 非创建人提交他人草稿（非管理员/值班长）   | 拒绝提交   | `api_submit_voucher` L464 |
| 非管理/值班长执行复核/关闭/撤销           | 403 无权限 | `role_required` 装饰器 |
| 已关闭/已撤销的单据再次操作               | 拒绝操作   | 各状态机判断          |

## 列表筛选
- **班次筛选**：早班 / 中班 / 晚班 / 自定义班次
- **处理人筛选**：按创建人、复核人、关闭人、当前处理人模糊匹配
- **状态筛选**：6 种状态单选
- **关键词搜索**：单据编号 / 收银员姓名 / 备注 / 原因 模糊匹配

## 差异汇总（首页顶部卡片）
- 待复核：数量 + 金额合计
- **未结数量**：草稿 + 待复核 + 已退回 + 复核通过 的总单数（核心指标）
- 复核通过 / 已关闭 / 已退回 / 已撤销 的数量与金额

## CSV 导入导出

### 导出字段
单据编号、状态、班次、班次日期、收银员、差异金额、原因、备注、创建人、创建时间、复核人、关闭人、退回说明、关闭说明、撤销人、撤销时间、关联原单、导入来源

### 导入要求
- 编码：UTF-8（带 BOM 也支持）或 GBK
- 必填字段：单据编号、班次、班次日期、收银员、差异金额
- 负金额必须有原因
- 不允许导入已关闭或已撤销状态的单
- 不允许重复导入相同单号
- 支持中文表头（即导出格式可以直接再次导入）

## 常见问题

**Q: 浏览器无法访问？**
A: 确认 `启动.bat` 窗口未关闭，且无端口冲突（默认 5000）。

**Q: 忘记密码？**
A: 使用 SQLite 工具（如 DB Browser for SQLite）打开 `data/shift_diff.db`，查看 `users` 表即可看到明文密码，或替换为新值。

**Q: 想恢复到上周五的数据？**
A: 停止服务 → 用上周五的备份 `shift_diff_YYYYMMDD.db` 覆盖当前文件 → 重启。

**Q: 如何新增用户？**
A: 管理员直接在 `users` 表中 INSERT：
```sql
INSERT INTO users (username, password, role, display_name)
VALUES ('cashier3', 'cashier123', 'cashier', '收银员小王');
```

**Q: 数据文件体积越来越大？**
A: 可定期导出 CSV 归档后，执行 `VACUUM` 命令压缩 SQLite 文件。
